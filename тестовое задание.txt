Создать приложение состоящие из 3 API сервисов

1) Сервис контрагентов
REST API
1. добавить контрагента
2. обновить контрагента
3. удалить контрагента
4. получить данные контрагента по Id
5. получить список контрагентов 

Данные
Тип ИП или ООО
если ИП, то ФИО, если ООО Название Фирмы
адресс регистрации

2) Сервис документов

API
1. добавить документ
2. обновить документ
3. удалить документ
4. получить данные документа по Id
5. получить список документов по конкретному контрагенту

Данные
Номер документа
Название документа
Контрагент (Данные контрагента на момент сохранения документа)

3) Аудит сервис

Закрытый сервис без апи, который при каждом изменении информации по контрагенту или документу пишет логи (эвенты изменений сущностей можно дата + текущее состояние в JSON формате)

Технологии:
ASP.CORE 2.2 или 3, БД для сервисов (MSSQL или PostgreSQL), для взаимодействия с бд использовать Dapper
Сервисы могут строиться по 3-х звенной или гексоганальной архитектуре.

Пожелание (опционально): Взаимодействие с сервисом аудита должно происходить ассинхронно (можно использовать RabbitMQ, Kafka или создать NotificationService как шину, в который сервис контрагентов и документов будут писать, а сервис аудита опрашивать)
